อธิบายการทำงานของระบบ Scan & Clean Junk 🧹 (ฉบับปรับปรุง - High Safety v2)
======================================================

ระบบนี้ถูกออกแบบมาเพื่อลบไฟล์ขยะอย่างปลอดภัยสูงสุด โดยมีการเพิ่มระบบป้องกันไฟล์สำคัญดังนี้:

1. ตำแหน่งที่เข้าไปจัดการ (Target Paths):
   ระบบจะเข้าไปตรวจสอบในพื้นที่ชั่วคราวหลักของ Windows:
   - User Temp (%TEMP%): โฟลเดอร์เก็บไฟล์ชั่วคราวของผู้ใช้งานปัจจุบัน
   - System Temp (C:\Windows\Temp): โฟลเดอร์เก็บไฟล์ชั่วคราวของระบบ Windows

2. ระบบความปลอดภัยขั้นสูง (Safety Measures):
   - 🌟 ระบบ Whitelist: ระบบจะไม่แตะต้องโฟลเดอร์ที่มีชื่อเกี่ยวข้องกับค่ายเกมและโปรแกรมสำคัญ เช่น Steam, Epic, Unity, Unreal, Hoyoverse, Discord และอื่นๆ แม้จะอยู่ในโฟลเดอร์ Temp ก็ตาม
   - 🌟 การลบแบบ Inside-Out (Top-down=False): ระบบจะสแกนลบจากไฟล์ด้านในสุดออกมา เพื่อให้การจัดการโฟลเดอร์เปล่าทำได้อย่างแม่นยำ
   - 🌟 ปลอดภัย 100% ด้วย os.rmdir: ระบบเปลี่ยนจากการลบทั้งโฟลเดอร์ เป็นการลบเฉพาะ "โฟลเดอร์ที่ว่างเปล่าจริงๆ" เท่านั้น หากยังมีไฟล์เกมหรือไฟล์ใดๆ เหลืออยู่ภายใน (เนื่องจากถูกใช้งานอยู่) ระบบจะไม่ลบโฟลเดอร์นั้นทิ้งเด็ดขาด

3. ข้อควรระวังในการใช้งาน:
   - การเข้าถึง System Temp จำเป็นต้องรันโปรแกรมด้วยสิทธิ์ "Administrator"
   - **ข้อแนะนำ**: แม้ระบบจะมีความปลอดภัยสูง แต่ยังคงแนะนำให้หลีกเลี่ยงการกด Clean Junk ในขณะที่กำลังโหลดเกม เพื่อป้องกันปัญหา Disk I/O ทำงานหนักจนเกมอ่านไฟล์ไม่ทัน

4. การคำนวณและสรุปผล:
   - ระบบจะคำนวณขนาดไฟล์เฉพาะไฟล์ที่ "ลบสำเร็จแล้วจริงๆ" เท่านั้น เพื่อความแม่นยำสูงสุด
   - ผลลัพธ์จะแสดงเป็นจำนวนไฟล์ที่ถูกกำจัด และขนาดพื้นที่ (MB) ที่ได้รับคืนมา

มาตรฐานความปลอดภัย:
- ด้วยระบบ Whitelist และการลบเฉพาะโฟลเดอร์เปล่า (Empty Dir Only) คุณจึงมั่นใจได้ว่าไฟล์สำคัญของเกมและแอปพลิเคชันจะปลอดภัยจากการถูกลบโดยไม่ตั้งใจครับ
